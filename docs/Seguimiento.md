# Seguimiento del Proyecto SPIRITBLADE

## Blog con Anuncios sobre el Desarrollo del Proyecto

El proyecto SPIRITBLADE mantiene un blog en Medium para comunicar avances, decisiones técnicas y hitos del desarrollo. Los posts incluyen actualizaciones sobre fases completadas, problemas resueltos y lecciones aprendidas.

### Publicaciones Principales

- **Blog**: URL: https://medium.com/@j.andres.2022/fase-1-tfg-5ecf33a800e3   

### Propósito del Blog

- **Transparencia**: Mantener informados a tutores, compañeros y stakeholders sobre el progreso.
- **Documentación**: Registrar decisiones técnicas y desafíos encontrados.
- **Aprendizaje**: Compartir conocimientos adquiridos durante el desarrollo.
- **Networking**: Posible atracción de colaboradores o feedback de la comunidad.

## GitHub Project para Gestión de Tareas

El proyecto utiliza GitHub Projects con un tablero Kanban para gestionar el desarrollo de manera ágil e incremental.

### Configuración del Proyecto

- **Nombre del Proyecto**: SPIRITBLADE Development Board
- **URL**: https://github.com/codeurjc-students/2025-SPIRITBLADE/projects
- **Tipo**: Kanban Board
- **Columnas**:
  - **Backlog**: Tareas pendientes por priorizar
  - **In Progress**: Tareas en desarrollo activo
  - **In Review**: Tareas completadas esperando revisión
  - **Done**: Tareas finalizadas y mergeadas
  - **Discarded**: Tareas canceladas o no implementadas

### Gestión de Tareas

#### Issues y Cards
- Cada tarea se representa como un GitHub Issue con:
  - **Título descriptivo** siguiendo formato: `feat<NumTask>: Descripción`

#### Flujo de Trabajo
1. **Creación**: Issues se crean desde el backlog o durante planificación
3. **Desarrollo**: Movidas a "In Progress"
4. **Revisión**: Pull Request creado, movida a "In Review"
5. **Cierre**: Merge a main, movida a "Done"
6. **Cancelación**: Movida a "Discarded" si no se implementa
